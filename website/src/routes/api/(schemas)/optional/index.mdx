# optional

Creates an optional schema.

```ts
const Schema = optional<TWrapped, TDefault>(wrapped, default_);
```

## Generics

- `TWrapped` <Property {...properties.TWrapped} />
- `TDefault` <Property {...properties.TDefault} />

## Parameters

- `wrapped` <Property {...properties.wrapped} />
- `default_` {/* prettier-ignore */}<Property {...properties.default_}/>

### Explanation

With `optional` the validation of your schema will pass `undefined` inputs, and if you specify a `default_` value, the schema will use it if the input is `undefined`.

> Note that `optional` does not accept `null` as an input. If you want to accept `null` inputs, use <Link href="../nullable/">`nullable`</Link>, and if you want to accept `null` and `undefined` inputs, use <Link href="../nullish/">`nullish`</Link> instead. Also, if you want to set a default value for any invalid input, you should use <Link href="../fallback/">`fallback`</Link> instead.

## Returns

- `Schema` <Property {...properties.Schema} />

## Examples

The following examples show how `optional` can be used.

### Optional string schema

Schema that accepts `string` and `undefined`.

```ts
const OptionalStringSchema = optional(string(), "I'm the default!");
```

### Optional date schema

Schema that accepts `Date` and `undefined`.

> By using a function as the default value, the schema will return a new `Date` instance each time the input is `undefined`.

```ts
const OptionalDateSchema = optional(date(), () => new Date());
```

### Optional entry schema

Object schema with an optional entry.

```ts
const OptionalEntrySchema = object({
  key: optional(string()),
});
```

### Unwrap optional schema

Use <Link href="../unwrap/">`unwrap`</Link> to undo the effect of `optional`.

```ts
const OptionalNumberSchema = optional(number());
const NumberSchema = unwrap(OptionalNumberSchema);
```
